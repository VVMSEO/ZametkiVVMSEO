# Заметки VVMSEO

Полноценное веб-приложение для записи идей и планирования дел, работающее прямо в браузере. Приложение полностью статичное (Vanilla JS, HTML, CSS) и использует Firebase для хранения данных и авторизации.

## Возможности
- **Авторизация через Google**: Безопасный вход и привязка заметок к вашему аккаунту.
- **Создание и редактирование**: Удобный редактор с поддержкой Rich Text.
- **Форматирование текста**: Полужирный, курсив, подчеркивание, зачеркивание, списки.
- **Вставка изображений**: Добавление картинок прямо в текст заметки (с автоматическим сжатием).
- **Отмена и повтор (Undo/Redo)**: Удобное управление изменениями.
- **Синхронизация в реальном времени**: Заметки мгновенно сохраняются в облаке (Firestore) и доступны на всех устройствах.
- **Закрепление**: Важные заметки можно закрепить вверху списка.
- **Горячие клавиши**: Создавайте новую заметку с помощью `Ctrl + Alt + N`.

## Технологии
- Vanilla JavaScript (ES Modules)
- HTML5 & CSS3 (без сторонних фреймворков)
- Firebase Authentication (Google Login)
- Firebase Firestore (База данных реального времени)

## Публикация на GitHub Pages

Приложение уже полностью подготовлено для хостинга на GitHub Pages, так как не требует сборки (Node.js/Webpack/Vite).

### Шаг 1: Экспорт кода
1. В правом верхнем углу AI Studio нажмите на иконку шестеренки (Settings) или меню экспорта.
2. Выберите **"Export to GitHub"** (или скачайте ZIP-архив и загрузите его в свой репозиторий на GitHub вручную).

### Шаг 2: Настройка GitHub Pages
1. Перейдите в ваш репозиторий на GitHub.
2. Откройте вкладку **Settings** (Настройки).
3. В левом меню выберите **Pages**.
4. В разделе "Build and deployment" -> "Source" выберите **Deploy from a branch**.
5. В разделе "Branch" выберите `main` (или `master`) и папку `/ (root)`. Нажмите **Save**.
6. Через пару минут ваш сайт будет доступен по ссылке вида `https://<ваш-логин>.github.io/<имя-репозитория>/`.

### Безопасность конфигурации Firebase
Файл `firebase-applet-config.json` содержит ключи для подключения к Firebase. **Публиковать эти ключи в открытом репозитории для статических сайтов (SPA) — это нормальная практика**, так как они нужны браузеру для подключения к базе данных. 

Безопасность ваших данных обеспечивается **Правилами безопасности Firestore (Security Rules)**, которые мы уже настроили. Они гарантируют, что никто не сможет прочитать или изменить чужие заметки, даже зная ключи проекта.

## Настройка Firebase (если вы создаете проект с нуля)
Если вы захотите перенести приложение на другой проект Firebase:
1. Создайте проект в [Firebase Console](https://console.firebase.google.com/).
2. Включите **Firestore Database** и **Authentication** (провайдер Google).
3. Зарегистрируйте веб-приложение и скопируйте конфигурацию в файл `firebase-applet-config.json`.
4. Установите правила безопасности Firestore (см. файл `firestore.rules`).
